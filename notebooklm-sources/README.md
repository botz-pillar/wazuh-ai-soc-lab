# NotebookLM Source Files

Ready-made documents to upload to NotebookLM (notebooklm.google.com). These create a "compliance and security brain" you can query in plain English.

## How to use

1. Go to [notebooklm.google.com](https://notebooklm.google.com)
2. Create a new notebook called "CloudVault Security"
3. Click "Add source" and upload all the files in this folder
4. Start asking questions

## What's included

| File | What it gives you |
|------|------------------|
| `soc2-trust-services-criteria.md` | The full SOC 2 framework — ask about any control requirement |
| `cloudvault-security-profile.md` | CloudVault Financial's complete security posture — tech stack, controls, gaps |
| `vendor-evaluation-guide.md` | How to evaluate vendor security — criteria, red flags, questions to ask |
| `incident-response-framework.md` | IR process based on NIST — phases, checklists, communication templates |

## Example queries

Once uploaded, try asking:

- "What does SOC 2 control CC6.1 require for a company like CloudVault?"
- "Based on CloudVault's infrastructure, what evidence would satisfy the access control requirements?"
- "I'm evaluating a SIEM vendor. What questions should I ask about data segregation?"
- "Walk me through the containment phase of incident response for a data exfiltration event"
- "Which SOC 2 controls relate to change management, and how is CloudVault doing on those?"
